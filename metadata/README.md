# Metadata (`metadata/`)

This folder contains all structured data artifacts used throughout the DIP-based AI image detection pipeline.

Rather than operating directly on raw image files after initial acquisition, the pipeline is driven entirely by metadata and derived data representations stored in this directory.

---

## Overview

The `metadata/` directory organizes the full data processing workflow into clearly defined stages:

1. Original dataset metadata
2. Data integrity and deduplication (hashes)
3. Preprocessed metadata
4. Train/test splits
5. Feature extraction
6. Feature vector construction
7. Model training artifacts
8. Final evaluation results

Each stage produces outputs that serve as inputs to later stages.

---

## Structure

```text id="g3j1p4"
metadata/
├── original/
├── hashes/
├── preprocessed/
├── splits/
├── features/
├── vectors/
├── models/
└── results/
```

---

## Stage Descriptions

### `original/`

Contains raw metadata generated during dataset acquisition.

* One CSV per data source (e.g., COCO, ImageNet, DiffusionDB)
* Includes basic information such as filenames and source identifiers
* Direct output of **Notebook 01 — Build Dataset**

---

### `hashes/`

Stores hash-based data used for integrity checks and duplicate detection.

* Global and per-source hash files
* Supports ensuring dataset uniqueness and consistency

---

### `preprocessed/`

Contains cleaned and standardized metadata.

* Normalized column structure across all datasets
* Consistent labeling (`real` vs `ai`)
* Direct output of **Notebook 02 — Preprocess Images**

---

### `splits/`

Defines the dataset partitions used for model training and evaluation.

* `combined_metadata.csv`
* `train_metadata.csv`
* `test_metadata.csv`

These files determine which samples belong to each subset and ensure reproducibility.

---

### `features/`

Contains extracted feature sets for each dataset subset.

* Gradient-based features
* Spatial features
* Frequency-domain features

Generated by the feature extraction notebooks.

---

### `vectors/`

Contains the final assembled feature vectors used as model inputs.

* Combines all feature groups into a single representation
* Separate files for training and testing datasets

---

### `models/`

Stores model-related artifacts produced during training.

* Model comparison results
* Cross-validation results
* Selected model configurations
* Trained model files (`.pkl`)
* Scaler object (`scaler.pkl`)

---

### `results/`

Contains final evaluation outputs.

* Confusion matrix data
* ROC curve points
* Final test metrics (CSV and JSON formats)

These represent the final performance of the selected model on the held-out test set.

---

## Design Philosophy

The `metadata/` directory is designed around the following principles:

* **Pipeline-driven structure**
  Each subfolder corresponds to a specific stage in the processing pipeline.

* **Reproducibility**
  All intermediate results are stored explicitly, allowing the pipeline to be resumed or inspected at any stage.

* **Separation from raw data**
  After initial dataset acquisition, processing is performed using metadata and derived features rather than raw images.

* **Traceability**
  Every output can be traced back to a prior stage, enabling debugging and analysis.

---

## Notes

* Large files (e.g., full datasets or large model artifacts) may be relocated to GitHub Releases if needed.
* This directory represents the backbone of the project’s data workflow.
* Not all stages must be recomputed if intermediate outputs are already available.
