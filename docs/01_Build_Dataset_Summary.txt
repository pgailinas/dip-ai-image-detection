Notebook 01 — Dataset Builder (Summary)

Purpose Build a standardized dataset from multiple image sources (real
and AI-generated), ensuring: - Fixed number of accepted images per
source (e.g., 3000) - Consistent preprocessing - Deduplication via
hashing - Structured metadata output (CSV)

Overall Workflow 1. Select target dataset 2. Load target-specific module
3. Configure dataset parameters 4. Initialize CSV + utilities 5. Load
source dataset 6. Extract candidate records 7. Process and filter images
8. Save images + metadata 9. Verify dataset integrity

Cell-by-Cell Summary

Cell 0 — Startup (Target-Agnostic) - Installs dependencies - Imports
required libraries (NumPy, PIL, etc.) - Sets up environment

Cell 1 — Notebook Summary - Describes purpose and structure of the
notebook - Acts as documentation header

Cell 2 — Select Target Dataset - Defines available dataset targets
(e.g., DiffusionDB, COCO, etc.) - User selects which dataset to build

Cell 3 — Load Target Module - Dynamically imports target-specific Python
module - Enables modular dataset handling

Cell 4 — Common Configuration - Sets parameters such as: - Target image
count (e.g., 3000) - Batch size - Random seed

Cell 5 — CSV Setup - Defines metadata schema: - filename - class_label
(ai / real) - source_dataset - resolution - Initializes CSV file

Cell 6 — Hash Utilities - Loads and maintains hash sets - Prevents
duplicate images: - Within dataset - Across datasets

Cell 7 — Filename & Counting Utilities - Tracks number of saved images -
Generates standardized filenames: [label][dataset][index].png

Cell 8 — Image Utilities - Handles preprocessing: - Resize to 256×256 -
Convert to grayscale - Ensures consistent format

Cell 9 — CSV Append Utility - Appends metadata rows to CSV - Keeps
dataset logging consistent

Cell 10 — Load Source Dataset - Calls target-specific loader - Retrieves
dataset (e.g., from Hugging Face)

Cell 11 — Preview Candidate Records - Samples candidate images -
Validates structure before full processing

Cell 12 — Process One Candidate - Core processing logic: - Load image -
Validate size - Normalize - Compute hash - Reject duplicates - Save
image + metadata

Cell 13 — Build Dataset - Main loop: - Iterates over candidate records -
Applies filtering and processing - Stops when target count reached

Cell 14 — Verify Saved Output - Checks: - CSV row count - Image count -
Ensures consistency

Cell 15 — Run Builder - Executes dataset build process

Cell 16 — Final Verification - Confirms: - Expected number of images
collected - No missing entries

Cell 17 — Final Inspection - Loads CSV into pandas - Allows quick
inspection and sanity checks

Key Design Features - Modular (target-specific .py files) -
Duplicate-safe (hash-based filtering) - Reproducible (fixed naming + CSV
tracking) - Scalable (batch processing + configurable limits) -
Consistent preprocessing (256×256 grayscale)
