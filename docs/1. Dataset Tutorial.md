# 1. Dataset Tutorial

## Overview

This section describes how the dataset used in this project is constructed, organized, and prepared for machine learning tasks. The dataset consists of both real and AI-generated images and is designed to support a reproducible pipeline for feature extraction and classification.

The workflow covers dataset acquisition, preprocessing, metadata construction, and the creation of balanced training and test splits. All stages are driven by metadata, ensuring consistent data handling and preventing leakage between datasets.

## Dataset Structure

The dataset is composed of multiple image sources representing both real and AI-generated content. Real images are drawn from established datasets, while AI-generated images are obtained from modern generative models.

- **Real datasets:** ImageNet_1K_256, MS_COCO_2017, OpenImages
- **AI-generated datasets:** DiffusionDB, SDXL Generated, MidJourney

Each image is associated with metadata describing the filename, class label (`real`, `ai`) and source dataset. This metadata-driven design allows all processing steps to operate without reliance on directory structure.

## Data Format

Images are processed into a standardized format suitable for feature extraction. Each image is represented as a NumPy array:

- Grayscale: `(H, W)`
- Color (if used): `(H, W, 3)`

Labels are categorical:

- `real`
- `ai`

This consistent representation ensures compatibility across all downstream processing stages.

## Dataset Splits

The dataset is divided into training and test subsets using metadata-based splitting.

- `train_metadata.csv` — training dataset
- `test_metadata.csv` — test dataset

Splits are:

- balanced across datasets and classes
- reproducible using a fixed random seed

All splits are defined at the metadata level, ensuring that images are never duplicated or moved between subsets.

## Workflow

This section prepares the dataset through a sequence of steps, beginning with dataset construction and ending with the creation of training and test splits.

- [01 Build Dataset](01_Build_Dataset.md) — construct dataset from source images  
- [02 Preprocess Images](02_Preprocess_Images.md) — standardize image format  
- [03 Combine and Split](03_Combine_and_Split.md) — create balanced train/test splits  

Each stage is implemented as a separate notebook and produces artifacts used in subsequent steps of the pipeline.

## Notes

- Dataset construction is optional if prebuilt datasets are already available
- All processing is metadata-driven to ensure reproducibility and consistency
- Preprocessed datasets are the default input for feature extraction
- Raw datasets are retained primarily for dataset construction and experimentation
- Dataset splits are defined at the metadata level to prevent data leakage between training and test sets
