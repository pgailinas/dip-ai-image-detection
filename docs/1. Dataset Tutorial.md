# 1. Dataset Tutorial

## Overview

This section describes how the image dataset used in this project is structured, prepared, and organized for machine learning tasks.

The dataset includes both real and AI-generated images and is designed to support a reproducible pipeline for feature extraction and classification.

---

## Objectives

- Define the structure of the dataset used in the project  
- Describe the preprocessing pipeline for image standardization  
- Explain the role of metadata in dataset management  
- Describe the formation of training and test splits  

---

## Dataset Structure

The dataset consists of:

- Real images:
  - ImageNet_1K_256  
  - MS_COCO_2017  

- AI-generated images:
  - DiffusionDB  
  - SDXL Generated  
  - MidJourney  

Each image is associated with metadata describing:
- filename  
- class label (`real`, `ai`)  
- source dataset  

---

## Data Format

- Inputs: image data represented as NumPy arrays `(H, W)` or `(H, W, 3)`  
- Labels: categorical values (`real`, `ai`)  

---

## Dataset Splits

The dataset is divided into:

- `train_metadata.csv` — training data  
- `test_metadata.csv` — test data  

Splits are:
- balanced across datasets  
- reproducible using fixed random seeds  

---

## Workflow Summary

This section prepares the dataset through three stages:

- Build dataset from source images  
- Preprocess images into a standardized format  
- Combine metadata and create train/test splits  

---

## Notes

- Dataset construction is optional if prebuilt datasets are available  
- Metadata-driven design ensures no data leakage  
- All downstream processing relies on metadata, not file structure
