## Overview

This section presents the **final training and evaluation stage** of the pipeline.

At this point, the dataset has been constructed, features have been extracted and normalized, and optimized classifier configurations have been identified. The goal of this section is to:

* train finalized models
* evaluate performance on unseen data
* analyze results and model behavior

This stage represents the transition from model development to **final performance assessment and interpretation**.

---

## Training and Evaluation Strategy

The full training process consists of two primary stages:

### 1. Final Model Evaluation

Trained classifiers are evaluated using a **held-out test dataset** that has not been used during training or validation.

* Models are applied to normalized test feature vectors
* Predictions and class probabilities are generated
* Performance metrics are computed, including:

  * Accuracy
  * Precision
  * Recall
  * F1-score
  * ROC-AUC

Additional evaluation artifacts include:

* confusion matrices
* ROC curve data
* comparative performance summaries

This ensures an **unbiased assessment of generalization performance**.

---

### 2. Further Analysis

Beyond standard evaluation, additional analyses are performed to better understand model behavior:

* **Feature-Level Analysis (11A)**
  Evaluates the predictive power of individual features

* **Source-Pair Analysis (11B)**
  Examines model performance across dataset combinations

These analyses provide deeper insight into:

* feature importance
* dataset sensitivity
* robustness of the DIP feature representation
