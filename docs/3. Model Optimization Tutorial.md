## Overview

This section focuses on **model selection, tuning, and training** using the DIP feature representation.

Following feature extraction and normalization, the objective is to identify effective classifier configurations and produce trained models that generalize well to unseen data.

The optimization process emphasizes:

* systematic comparison of candidate classifiers
* controlled hyperparameter tuning
* robust validation using cross-validation

---

## Optimization Strategy

Model optimization is performed in three structured stages:

### 1. Classifier Selection

A range of candidate classifiers is evaluated using normalized feature vectors.

* Multiple model types are considered (e.g., SVM, MLP, ensemble methods)
* Performance is measured using **stratified k-fold cross-validation**
* Metrics include:

  * Accuracy
  * Precision
  * Recall
  * F1-score
  * ROC-AUC

The goal is to identify the **top-performing classifiers** rather than prematurely selecting a single model.

---

### 2. Hyperparameter Tuning

Top-performing classifiers are refined using **controlled hyperparameter tuning**.

* Small, targeted parameter grids are used
* Tuning focuses on key parameters (e.g., regularization, kernel parameters, network architecture)
* Cross-validation is used to evaluate configurations

This approach balances:

* improved performance
* computational efficiency
* interpretability

---

### 3. Final Model Training

Selected classifiers are retrained using the **full training dataset**.

* Final models use optimized hyperparameters
* Training uses all available training data
* Models are saved for reproducible evaluation

Two classifiers are retained:

* **RBF Support Vector Machine (RBF SVM)**
* **Multi-Layer Perceptron (MLP)**

This dual-model approach improves robustness and enables comparative evaluation.

---

## Validation Approach

Model validation is performed using:

* **Stratified k-fold cross-validation**

  * Maintains class balance across folds
  * Produces stable performance estimates

* **Multiple evaluation metrics**

  * Captures different aspects of model performance

Validation is conducted **only on training data**, ensuring the test dataset remains independent.
