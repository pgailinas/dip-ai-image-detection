# 3. Model Optimization Tutorial

## Overview

This section focuses on model selection, tuning, and training using the DIP feature representation developed in the previous stage. The goal is to identify effective classifier configurations and produce trained models that generalize well to unseen data.

The optimization process emphasizes systematic comparison of candidate classifiers, controlled hyperparameter tuning, and robust validation using cross-validation. These steps transform feature vectors into trained models that are ready for final evaluation.

## Optimization Strategy

Model optimization is performed through a structured process that includes classifier selection, hyperparameter tuning, and final model training.

A range of candidate classifiers is first evaluated using normalized feature vectors. Multiple model types, including Support Vector Machines (SVM), Multi-Layer Perceptrons (MLP), and tree-based models, are compared using stratified k-fold cross-validation. Performance is assessed using metrics such as accuracy, precision, recall, F1-score, and ROC-AUC, allowing the most effective models to be identified.

The top-performing classifiers are then refined through controlled hyperparameter tuning. Rather than performing exhaustive searches, the process uses small, targeted parameter grids focused on key model characteristics such as regularization strength, kernel parameters, and network architecture. Cross-validation is used throughout to ensure that performance improvements are consistent and not the result of overfitting.

Once optimal configurations are identified, the selected classifiers are retrained using the full training dataset. This ensures that all available training data contributes to the final models. Two classifiers are retained for further evaluation: an RBF Support Vector Machine and a Multi-Layer Perceptron. Using multiple models improves robustness and allows performance to be compared across different modeling approaches.

Validation is performed exclusively on the training data using stratified k-fold cross-validation, which maintains class balance across folds and provides stable performance estimates. The test dataset remains completely independent and is reserved for final evaluation.

## Workflow

This section defines the model optimization pipeline, progressing from initial classifier evaluation to fully trained models.

- [07 Classifier Selection](07_Classifier_Selection.md) — evaluate candidate models using cross-validation
- [08 Train Two Classifiers](08_Train_Two_Classifiers.md) — train selected classifiers with optimized configurations
- [09 Validate and Tune Two Models](09_Validate_and_Tune_Two_Models.md) — refine performance and compare models

Each stage builds on the previous step to progressively improve model performance and produce trained models ready for final evaluation.

## Notes

- Cross-validation is used for model selection and tuning, while the test dataset is reserved for final evaluation
- Hyperparameter tuning is intentionally constrained to maintain interpretability and efficiency
- Multiple classifiers are retained to improve robustness and support comparative analysis
- Final models are trained using the full training dataset to maximize available information
