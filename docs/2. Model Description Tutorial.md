## Overview

This section describes the **modeling pipeline** used for AI-generated image detection based on Digital Image Processing (DIP) features.

Unlike end-to-end deep learning approaches, this project uses a **feature-engineering + classical machine learning framework**, where carefully designed image descriptors are extracted and used as input to a classifier.

The pipeline consists of the following stages:

1. **Feature Extraction**

   * Gradient-based features (edge and orientation structure)
   * Spatial features (intensity, entropy, texture)
   * Frequency-domain features (spectral energy and distribution)

2. **Feature Vector Construction**

   * Individual feature groups are combined into a fixed-length DIP feature vector

3. **Normalization**

   * Features are scaled using a transformation fit on training data

4. **Classifier Selection and Training**

   * Multiple classifiers are evaluated
   * Top-performing models are selected and trained

5. **Evaluation**

   * Models are evaluated using cross-validation and independent test data

This structured approach emphasizes **interpretability, modularity, and generalization across datasets**.

---

## Model Representation

Each image is represented by a **fixed-length DIP feature vector** derived from multiple complementary feature groups.

* Gradient-based features capture **edge structure and directional patterns**
* Spatial features capture **intensity distribution and texture**
* Frequency-domain features capture **spectral characteristics**

These features are concatenated into a unified vector used as input to the classifier.

---

## Classifier Design

The classification stage operates on normalized feature vectors and uses standard supervised learning methods.

### Candidate Models

A range of classifiers is evaluated, including:

* Support Vector Machines (SVM)
* Multi-Layer Perceptron (MLP)
* Tree-based models (e.g., Random Forest)

### Final Model Selection

Based on cross-validation performance, the **top two classifiers** are selected and trained:

* **RBF SVM**
* **Multi-Layer Perceptron (MLP)**

This dual-model approach:

* reduces sensitivity to small performance differences
* enables comparison across model types
* improves robustness of conclusions

---

## Training Strategy

* Training data is used for:

  * model fitting
  * cross-validation

* A **stratified k-fold cross-validation** framework is used for:

  * model comparison
  * hyperparameter tuning

* Final models are trained on the **full training dataset**

---

## Evaluation Strategy

* A **held-out test dataset** is used for final evaluation

* Metrics include:

  * Accuracy
  * Precision
  * Recall
  * F1-score
  * ROC-AUC

* Additional analyses include:

  * feature-level evaluation
  * source-pair generalization studies
