# 5. Basic Fine-Tuning Tutorial

## Overview

This section demonstrates how model performance can be improved through controlled fine-tuning of classifier configurations. Rather than performing iterative retraining of a pre-trained model, fine-tuning in this project is achieved through systematic hyperparameter selection and model comparison using the DIP feature representation.

The goal is to refine baseline performance by identifying improved configurations while maintaining interpretability and avoiding overfitting.

## Fine-Tuning Approach

Fine-tuning is performed by evaluating variations of model configurations using the training dataset. Candidate configurations are defined by adjusting key hyperparameters such as regularization strength, kernel parameters, and network architecture.

Each configuration is evaluated using stratified k-fold cross-validation, allowing performance to be compared across multiple metrics. This approach ensures that improvements are consistent and not dependent on a particular data split.

The best-performing configuration is selected and retrained using the full training dataset. The resulting model is then evaluated on the test dataset to measure improvement over the baseline established in the previous section.

This process demonstrates how controlled adjustments to model parameters can improve performance without requiring changes to the underlying feature representation.

## Notes

- Fine-tuning is performed through hyperparameter selection rather than iterative retraining loops
- Cross-validation is used to evaluate candidate configurations on training data
- The test dataset is used only for final evaluation of the tuned model
- Improvements are measured relative to the baseline performance established in the testing stage
