## Purpose

This notebook combines the three feature groups:

* Gradient features (04A)  
* Spatial features (04B)  
* Frequency-domain features (04C)  

into a unified **feature vector representation** for each image.

These combined feature vectors form the complete **25-dimensional DIP feature set** used for classification.

---

## Inputs

The notebook expects the following feature CSV files:

* `metadata/features/train_gradient_features.csv`
* `metadata/features/train_spatial_features.csv`
* `metadata/features/train_frequency_features.csv`

* `metadata/features/test_gradient_features.csv`
* `metadata/features/test_spatial_features.csv`
* `metadata/features/test_frequency_features.csv`

These files are generated by the three preceding notebooks.

## Outputs

The following combined feature vector datasets are generated under `metadata/vectors/`:

* `metadata/vectors/train_feature_vectors.csv`
* `metadata/vectors/test_feature_vectors.csv`

Each dataset includes:

* Metadata columns:
  * `filename`
  * `class_label`
  * `source_dataset`
  * `subset`

* Combined feature columns (25 total)

⚠️ These files are written to the Colab runtime environment and are **not automatically persisted to Google Drive**.

---

## Main Tasks

* Load gradient, spatial, and frequency feature datasets
* Verify row alignment across feature groups
* Merge feature groups using metadata keys
* Construct unified feature vectors
* Validate column structure and feature count
* Save combined feature vector CSV files

---

## Processing Workflow

This notebook executes a structured sequence of steps:

1. **Load Feature Groups**  
   Gradient, spatial, and frequency feature CSV files are loaded for the selected subset.

2. **Data Alignment Verification**  
   Ensures that all feature groups contain identical rows in the same order.

3. **Feature Merging**  
   Feature groups are merged using metadata columns:
   * `filename`
   * `class_label`
   * `source_dataset`
   * `subset`

4. **Feature Vector Construction**  
   All feature columns are combined into a single dataset.

5. **Validation**  
   The final feature vector dataset is validated for:
   * Correct row count
   * Correct number of feature columns
   * No missing values

6. **Output Generation**  
   The combined feature vectors are saved for downstream processing.

---

## Notes and Design Choices

* **Metadata-driven merging:**  
  Feature groups are aligned using shared metadata rather than positional assumptions.

* **Strict dataset integrity:**  
  Train and test sets remain completely separate throughout processing.

* **Modular design:**  
  Each feature group is independently computed and then combined.

* **No recomputation:**  
  This notebook strictly combines existing features — no image processing occurs here.

* **Consistent column ordering:**  
  Feature columns are organized in a fixed, deterministic order for downstream use.
